# 数据结构课程设计周报（第 5 周）

## 一、本周基本信息

- 项目名称：基于智能体的个性化旅游系统设计与开发
- 周次：第 5 周
- 阶段定位：项目启动与基础版本搭建阶段
- 本周目标：
  - 明确课程设计题目范围与核心功能模块
  - 完成技术选型与系统总体架构设计
  - 搭建前后端基础开发环境
  - 实现部分核心功能原型，为后续扩展奠定基础

## 二、本周完成情况

### 1. 完成了项目需求梳理与模块拆分

结合课程讲义中“个性化旅游系统”的要求，对项目进行初步需求分析，确定本系统围绕以下核心模块展开：

- 旅游推荐模块
- 旅游路线规划模块
- 旅游日记管理与交流模块
- 协作行程模块
- 美食推荐模块
- 用户登录与基础状态管理模块

在此基础上，对系统进行了前后端分层设计，明确了页面层、接口层、业务层、数据层之间的职责划分。

### 2. 完成了系统技术选型与开发环境初始化

本项目当前采用前后端分离架构，技术栈如下：

- 前端：Vue 3 + Vite + Vue Router + Pinia + Axios + Element Plus
- 后端：Java 17 + Spring Boot + Spring Data JPA
- 数据层：MySQL
- 检索层：Elasticsearch（用于旅游日记全文检索）

本周已完成前端工程和后端工程初始化，建立了项目的目录结构，形成了可以继续并行开发的基础框架。

### 3. 完成了前端整体页面框架与路由结构搭建

已完成前端主应用框架搭建，并建立以下页面路由：

- 首页
- 登录页
- 目的地推荐页
- 路线规划页
- 旅游日记页
- 协作行程页

同时实现了统一导航栏、页面容器布局和基础视觉风格，使系统具备初步演示能力。

### 4. 完成了后端基础接口设计与主要控制器搭建

本周完成了后端控制器与接口雏形搭建，主要包括：

- 用户登录接口
- 目的地查询与 Top-K 推荐接口
- 路线规划接口
- 日记列表、创建、全文检索接口
- 行程列表与创建接口
- 美食推荐接口

通过接口分层设计，后续可以继续在当前基础上补充算法和业务逻辑，而无需推翻整体结构。

### 5. 完成了部分核心算法的初步实现

围绕课程设计中的重点算法要求，本周已开始实现以下核心内容：

- 推荐模块：采用小顶堆实现 Top-K 非完全排序，避免对全部数据进行完整排序
- 路线规划模块：采用基于图结构的最短路径算法，实现单起点单终点的路径规划雏形
- 日记检索模块：完成全文检索接口结构设计，为后续文本检索与优化打下基础

虽然当前仍处于原型阶段，但算法实现方向与课程要求保持一致。

### 6. 完成了基础数据模型与样例数据准备

已建立部分核心实体模型，包括：

- 目的地
- 美食
- 日记
- 行程
- 道路节点
- 道路边
- 场所设施

同时实现了项目启动时的样例数据初始化，用于支撑基础功能联调和页面演示。

## 三、本周阶段成果

截至第 5 周末，项目已经完成了“从无到有”的基础版本搭建，形成了以下阶段成果：

- 确定系统方向和核心模块
- 搭建前后端分离架构
- 建立主要页面与主要接口
- 开始将课程要求中的数据结构与算法融入系统实现
- 形成了可继续扩展的原型系统

整体来看，本周工作的重点是“搭框架、立结构、通主链路”，为中期检查前的功能完善提供了较好的基础。

## 四、本周存在的问题

### 1. 当前数据规模较小

虽然已经准备了样例数据，但与课程讲义要求相比，当前数据量仍明显不足，尚不能体现真实旅游场景下的算法效果。

### 2. 若干核心功能尚未闭环

目前已经完成基础框架和部分功能，但多目标路径规划、场所查询、地图展示、多用户真实化等模块仍未完成。

### 3. 文档沉淀仍不充分

本周主要精力集中在架构与代码搭建上，需求分析、数据结构说明、接口文档、测试说明等材料还需要继续补充。

## 五、本周问题处理情况

针对上述问题，本周已形成以下应对思路：

- 先确保主功能模块具备演示原型，再逐步补齐课程要求中的完整功能
- 在现有架构不变的前提下，逐步扩展数据量和算法实现深度
- 后续开发过程中同步沉淀文档，避免结项前集中补材料带来的风险

第 6 周计划围绕“功能补强与可用性提升”展开，重点任务如下：

4. 完善路线规划模块的策略参数设计
5. 补充系统架构说明与验收对照材料
6. 梳理当前项目与课程讲义要求之间的差距，形成后续开发清单

## 七、本周总结

第 5 周是本项目正式启动的第一周，团队工作重点放在“系统搭建”和“主线贯通”上。当前已经形成一个具备实际开发基础的前后端项目，后续将在此基础上继续补足算法深度、数据规模和验收功能，逐步向课程设计最终要求靠拢。
